File Share exclusion revamp

Topic · Microsoft · Graph Connectors

Problem statement

File share connectors were missing content from valid file types because of overly restrictive exclusion rules. Admins couldn't easily exclude unwanted file extensions on a per-tenant basis, forcing a one-size-fits-all policy. Customers had to settle for incomplete search indexes.

Context & constraints

The feature required careful rollout: migration of existing connections, validation of new behavior, and rollback capability if issues surfaced. Large customers (Napier City Council with 5M items, Equinor) depended on this change to adopt the product.

Approach

  • Feature engineering: Revamped exclusion rule logic to be tenant-configurable and transparent
  • Safety: Built migration job to backfill existing connections with new rules and rollback job for fast recovery
  • Validation: Wrote comprehensive test suite to cover edge cases and prevent regression
  • Monitoring: Tracked item count gain in dashboard during and after rollout

Key decisions & tradeoffs

  • Tenant-scoped configuration: Allowed admins to customize exclusion rules per tenant rather than forcing all changes globally
  • Migration + rollback: Prioritized safety over speed — batch migration of existing connections plus rollback job gave ops confidence to proceed
  • Test-driven migration: Invested in test coverage upfront to avoid customer-impacting bugs during rollout

Impact

Feature status: Coded and rollout underway (Nov 2023 onwards). Expected to increase indexed items across all file share connectors.

Customer adoption: Unblocked high-value customers (Napier City Council with 5M+ items, Equinor) who were seeing incomplete indexes due to overly strict rules. Migration enabled smooth tenant-by-tenant rollout without downtime.
